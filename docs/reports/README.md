# Rapports - Boxing Round Splitter

Ce dossier contient les rapports de session et les analyses générés pendant le développement du projet.

## Structure

```
reports/
├── README.md                    # Ce fichier
├── SESSION_YYYY-MM-DD_xxx.md    # Rapports de session
└── nom_analyse_report.md       # Rapports d'analyse
```

## Types de Documents

### Rapports de Session
Format : `SESSION_YYYY-MM-DD_description.md`

Contenu typique :
- Date de la session
- Objectifs atteints
- Résultats concrets
- Prochaines étapes

### Rapports d'Analyse
Format : `nom_analyse_report.md`

Contenu typique :
- Description de l'analyse
- Méthodologie utilisée
- Résultats et conclusions
- Recommandations

## Règles

1. **Ne pas modifier** les rapports après création (ils représentent l'historique)
2. **Un rapport par session** pour maintenir la traçabilité
3. **Formatage clair** avec en-têtes et listes pour faciliter la lecture

## Historique

Consultez les fichiers individuels pour voir les détails de chaque session ou analyse.
